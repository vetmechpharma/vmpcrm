*   **MR Ordering System (MR & Admin):**
    *   MRs can place orders on behalf of customers. The order form in the MR app must match the functionality of the main CRM's order form, including item search, text-based quantity input (e.g., 10+5), an out-of-stock toggle, and notes.
    *   Orders placed by MRs should appear in the main admin order list, clearly marked as originating from an MR.
    *   MRs can request to cancel an order, which an admin must then approve or reject.
*   **PWA & Offline Support:** The entire MR App must be an installable PWA that works offline. Data created offline (like new orders or visits) should sync automatically when the connection is restored.
